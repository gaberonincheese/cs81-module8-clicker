# Cookie Clicker Reflection

## What helped you understand DOM interaction best?

I think having to actively look up how to do specific things with HTML helped me better understand how I'm supposed to interact and manipulate the DOM. I had a lot of fun learning what I could and couldn't do, and I'm excited to see what else I can get up to with more practice. 

## How did you choose your milestone messages?

I like dogs so I decided to imagine a scenario where the user collects them and eventually succumbs to the huge mass of dogs they've accumulated. I think the people that own large properties and provide safe homes to many previously stray / abandoned dogs are cool.

## What challenge or bug surprised you?

I struggled trying to implement audio initially. At first, I created <audio></audio> tags with sources for my chosen audio, but found that wasn't working as intended (I wasn't sure how to initialize a variable with those values). Eventually, I was able to get it working by making a variable and initializing it with new Audio. That allowed me to use the .play() function on my audio variables.

## What personal twist did you add?

I added an array of random dog sounds. The code randomly selects a sound from this array and plays it when a milestone is hit. I'm considering returning to this code to see if I can add more variety in the imagery, too.

## What real-world app uses this kind of interaction?

Nearly every real-world app I can think of benefits from the mechanics that power a Cookie Clicker type game. For example, a video game like Hollow Knight could not run without reacting to player input when a button is pressed. Learning how to implement this is foundational to establishing responsive web pages.
